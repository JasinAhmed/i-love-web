Learning Journal – HTML Code Review

  • Situatie: Ik gaf feedback op de HTML-code van een team en heb hun werk gereviewd.
  • Feedback ontvangen: Mijn eigen HTML had foutjes, zoals code die beter in JavaScript gezet kon worden.
  • Wat geleerd: HTML moet semantisch en schoon blijven. Functionele logica hoort in JS, niet in HTML.
  • Toepassing: Voortaan check ik mijn code met een checklist en let ik beter op de scheiding tussen HTML, CSS en JS.

Studievragen

  1. Wat is het nut van een breakdown schets?
  2. Waar moet je op letten als je een breakdown schets maakt?
  3. Hoe kan ik CSS en HTML combineren om een volledig responsive visitekaartje te maken?
